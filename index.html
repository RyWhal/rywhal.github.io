Math Functions

Here are some math functions... Its pretty self explanitory

add = (a, b)->\
    a + b
subtract = (a, b)->\
    a - b
divide = (a, b)->\
    a / b
multiply = (a, b)->\
    a * b

console.log "1 + 2 = "+add(1, 2)

Using Heregexes

A "heregex" is essentially your basic regex, except that hergexes can contain comments and white space. An example of using a heregex might look something like this:

#define your URLs (some that work, some that dont)
urls= [
    "http://rywhal.github.io"
    "https://google.com"
    "htp://something.com"
    "http://slur.pe"
    "soemthing.com"
    "thisisastring"
]

#define the regex pattern
pattern= ///
    https?:
    #see? you can have comments in here
    [a-zA-Z0-9-.]*
    /.*
///

#loop through URLs and validate against pattern
console.log "Valid URLs:"
for url in urls
    if url.match(pattern) != null
        console.log url

This scripts output:

$ coffee heregexes.coffee
Valid URLs:
http://rywhal.github.io
https://google.com
http://slur.pe

Chaining Operators

chaining operators together allows a lot of saved keystrokes in coffee.

in coffee:

mynumber=34

if 25 <= mynumber <= 50
    console.log "its between 24 and 50"
else
    console.log "its greater than 50 or less than 24"

The same operators in JS would look like this:

if ((24 <= mynumber && mynumber <= 50)){
...
}

One of these two is clearly less work :D

Making Classes and Functions

So everything is pretty plain language created in coffee-script. Creating a class is as easy as "class ClasName" and giving it a constructor with some attributes.

A function is easily created with a statement like "function_name: ->". The preceding indented lines are part of that function. Functions have implicit returns, too!

uncompiled info_print.coffee

class Info

    constructor: (@attributes)->

    printInfo: ->
        console.log "Name: #{@attributes.name}"
        console.log "Title: #{@attributes.title}"
        console.log "DOB: #{@attributes.dob}"
        if @attributes.salary
            console.log "Salary: #{@attributes.salary}"
        else
            console.log "Salary: Unknown"

ryan = new Info
    name: "Ryan"
    title: "Tech"
    dob: new Date(1988, 2, 2)
    salary: "$1,000,000"

ross = new Info
    name: "Ross S."
    title: "VP"
    dob: new Date(1985, 7, 15)

ryan.printInfo()
console.log " "
ross.printInfo()

after its compiled and run:

$ node info_print.js
Name: Ryan
Title: Tech
DOB: Wed Mar 02 1988 00:00:00 GMT+0000 (UTC)
Salary: $1,000,000

Name: Ross S.
Title: VP
DOB: Thu Aug 15 1985 00:00:00 GMT+0000 (UTC)
Salary: Unknown

String interpretation

Coffee allows you to cleanly and easily concatenate strings together and within each other. for example:

make = "Toyota"
model = "Camry"
year = 2012

concat= "My car is a #{year} #{make} #{model}"

console.log concat

In javascript, the concatenation would look something like this:

  make = "Toyota";
  model = "Camry";
  year = 2012;
  concat = "My car is a " + year + " " + make + " " + model;

Clearly coffee-script allows you to do this in a much cleaner manner.

Hello World

Turns out a hello world script is pretty easy in coffee script. It's easy to see how coffee script can save you time when you compare the .coffee and .js files.

uncompiled hello.coffee:

hello = "Hello, World"
console.log hello

compiled into js:

(function() {
  var hello;

  hello = "Hello, World";

  console.log(hello);

}).call(this);
